Props:
- `predictionsByDate: Record<string, Prediction[]>` — grouped predictions
- `months: MonthData[]` — array of 10 month objects for March-December 2026

State:
- `selectedDate: string | null` — which date's detail panel is open
- `highlightedWeek: string[] | null` — which high-risk dates in a week are highlighted

`handleDateClick(date: string, isHighRisk: boolean)`:
- If `isHighRisk`: set `selectedDate` to date, clear `highlightedWeek`
- If NOT high-risk: clear `selectedDate`, compute the Sun-Sat week containing `date`, find all dates in that week that ARE in `predictionsByDate`, set those as `highlightedWeek`

Week computation helper `getWeekDates(dateStr: string): string[]`:
- Parse the ISO date string to a Date object
- Find Sunday of that week: `day - day.getDay()`
- Generate 7 ISO strings (Sun through Sat) for that week
- Return the array

Render:
- Outer: `<div className="px-4 py-8 md:px-8">`
- Title: `<h2 className="text-lg font-semibold mb-4">2026 Earthquake Risk Calendar</h2>`
- Month grid container: `<div className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-6">` — responsive columns per UI-SPEC
- Map over `months` array, for each month:
  - Compute cells: create DateCell array for each day of the month plus leading padding cells. Day 1's weekday determines how many empty cells precede it. For each date cell, `isHighRisk = date in predictionsByDate`.
  - Render `<MonthGrid label={month.label} cells={cells} predictionsByDate={predictionsByDate} selectedDate={selectedDate} highlightedWeek={highlightedWeek} onDateClick={handleDateClick} />`
- Conditional DetailPanel: `{selectedDate && predictionsByDate[selectedDate] && <DetailPanel predictions={predictionsByDate[selectedDate]} date={selectedDate} onClose={() => setSelectedDate(null)} />}`

```typescript
import { loadPredictions, groupPredictionsByDate } from '@/lib/predictions'
import { CalendarInteractive } from '@/components/CalendarInteractive'

// Hardcoded month definitions for March-December 2026
// (only 10 months, fixed range — hardcoded is clearer per RESEARCH.md)
const MONTHS = [
  { year: 2026, month: 2, label: 'March 2026' },
  { year: 2026, month: 3, label: 'April 2026' },
  { year: 2026, month: 4, label: 'May 2026' },
  { year: 2026, month: 5, label: 'June 2026' },
  { year: 2026, month: 6, label: 'July 2026' },
  { year: 2026, month: 7, label: 'August 2026' },
  { year: 2026, month: 8, label: 'September 2026' },
  { year: 2026, month: 9, label: 'October 2026' },
  { year: 2026, month: 10, label: 'November 2026' },
  { year: 2026, month: 11, label: 'December 2026' },
]

export default async function CalendarPage() {
  const predictions = await loadPredictions()
  const predictionsByDate = groupPredictionsByDate(predictions)

  return (
    <CalendarInteractive
      predictionsByDate={predictionsByDate}
      months={MONTHS}
    />
  )
}
```

Key points:
- `month` values are 0-indexed JavaScript Date months (2=March, 11=December) — this is what `new Date(year, month, 1)` expects in CalendarInteractive's cell computation
- NO `'use client'` directive — this is a Server Component
- NO `fetch()` call — data loaded via `fs.readFile` inside `loadPredictions()`
- predictionsByDate is a plain Record (serializable as props to client component)
