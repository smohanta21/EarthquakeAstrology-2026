## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Calendar cell date number padding (`p-1`), inline label gaps |
| sm | 8px | Row gaps within calendar grid (`gap-2`), tight list spacing |
| md | 16px | Default element spacing, panel section padding |
| lg | 24px | Month block bottom margin, panel top padding |
| xl | 32px | Page horizontal padding, section breaks |
| 2xl | 48px | Major vertical separation between calendar and footer |
| 3xl | 64px | Page-level top/bottom padding |

Exceptions:
- Calendar cell touch target: `aspect-square` cells — minimum rendered size is ~44px at mobile breakpoint (375px / 7 cols = ~53px). No explicit override needed; grid math satisfies WCAG 2.5.5 naturally.
- Detail panel width: 384px (Tailwind `w-96`) — fixed, not on the 8-point token scale. This is a panel-specific structural dimension.

Source: RESEARCH.md code examples use `gap-1`, `p-1`, `mb-8`, `px-4 py-3`, `px-6 py-4`, `p-6` — all multiples of 4. Standard 8-point scale confirmed.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Region list entries in detail panel, aspect strings, methodology prose |
| Label | 12px | 400 (regular) | 1.4 | Calendar date numbers, day-of-week headers (Sun–Sat), metric labels |
| Heading | 18px | 600 (semibold) | 1.2 | Detail panel date heading, month name headings, page section titles |
| Display | 28px | 700 (bold) | 1.1 | Risk score number in detail panel (e.g., "0.73") |

Notes:
- Exactly 4 sizes, 2 effective weights (regular 400, semibold/bold 600–700). Bold (700) is used only for the single numeric risk score display — not a general weight.
- Font family: Inter (via `next/font/google`). Applied to `<html>` element via className in root layout.
- Uppercase label variant: `text-xs font-semibold text-gray-500 uppercase tracking-wide` — used for "Top Planetary Aspects" section header in detail panel (12px semibold).

Source: RESEARCH.md code examples; Claude's discretion per CONTEXT.md.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` (white) / `#f9fafb` (gray-50) | Page background, normal/no-prediction calendar cells, detail panel background |
| Secondary (30%) | `#f3f4f6` (gray-100) / `#e5e7eb` (gray-200) | Calendar grid background, nav border, methodology table rows, day-header row |
| Accent (10%) | `#fb923c` (orange-400) | High-risk calendar cell backgrounds ONLY |
| Risk bar fill | `#f97316` (orange-500) | Risk score progress bar fill in detail panel ONLY |
| Disclaimer surface | `#fffbeb` (amber-50) | Disclaimer banner background |
| Disclaimer border | `#fde68a` (amber-200) | Disclaimer banner bottom border |
| Destructive | not applicable | No destructive actions in this phase |

Accent reserved for:
1. High-risk date calendar cell background (`bg-orange-400`) — the single most important visual signal
2. Risk score progress bar fill in detail panel (`bg-orange-500`) — reinforces the same risk color in context

Week highlight treatment (clicking a normal date): `ring-2 ring-orange-400` outline on high-risk cells in the same week. Uses accent color as a ring/outline variant — not a fill — to distinguish "contextual highlight" from "direct selection".

Text on accent cells: `text-white` (white, 12px date number) — ensures sufficient contrast on orange-400 background (contrast ratio ~3.5:1 for large text; acceptable for 12px bold; use `font-medium` on date number to ensure legibility).

Source: RESEARCH.md code examples use `bg-orange-400`, `bg-orange-500`, `bg-amber-50`, `border-amber-200`. Claude's discretion per CONTEXT.md.

## Interaction Contract

### Calendar Cell — High-Risk Date
- **State:** Background `bg-orange-400`, date number `text-white text-xs font-medium`
- **Hover:** `hover:bg-orange-500 cursor-pointer` — slight darkening, pointer cursor
- **Click:** Opens DetailPanel (sets `selectedDate`); clears `highlightedWeek`
- **When panel open:** Selected cell gets `ring-2 ring-orange-600` to show active selection

### Calendar Cell — Normal Date (no prediction)
- **State:** Background `bg-gray-50`, date number `text-gray-700 text-xs`
- **Hover:** `hover:bg-gray-100 cursor-pointer`
- **Click:** Computes week boundaries from clicked date; sets `highlightedWeek` to all high-risk dates in that 7-day window (Sun–Sat); clears `selectedDate`
- **Week highlight active:** High-risk cells in week get `ring-2 ring-orange-400` outline; cleared on any subsequent interaction

### Calendar Cell — Empty Padding Cell
- **State:** Transparent background, no content, no cursor
- **Interaction:** None (not clickable)

### Detail Panel
- **Entry animation:** `translate-x-0` from `translate-x-full`; `transition-transform duration-300 ease-in-out`
- **Exit:** Reverse translate. Triggered by: outside-click overlay, clicking another date, or explicit close
- **Layout:** Fixed right-0, full height, `w-96` (384px), `bg-white shadow-xl`, `overflow-y-auto`, `z-50`
- **Outside-click overlay:** Full-screen `fixed inset-0 -z-10` div with `onClick={onClose}`; sits behind panel, above calendar
- **Panel sections (top to bottom):**
  1. Date heading — 18px semibold
  2. Max risk score — 28px bold + orange progress bar (100% width = risk_score 1.0)
  3. Regions list — sorted by risk_score descending, all entries shown; format: "[Country] — lat [N], lon [N]"; 14px regular
  4. Top Planetary Aspects — 12px uppercase semibold section label + bulleted list of aspect strings (underscore replaced with space); 14px regular list items

### Methodology Page
- Static route `/methodology`
- Reachable via "Methodology" link in nav bar
- Metrics displayed in a data table:
  - Model: XGBClassifier
  - F1 Score: 0.002774
  - MCC: 0.001363
  - Threshold: (from eval_report.json)
  - Eval split: 2010-01-01
  - Confusion matrix: TP / FP / FN / TN in a 2×2 table or 4-cell grid
- Explanatory prose: Claude's discretion; must be honest about low metric scores (F1 ~0.003 is near-zero — describe as experimental/exploratory, not as a validated forecasting system)
- Back to calendar: nav bar "Earthquake Astrology 2026" title links to `/`

### Disclaimer Banner
- Position: Top of every page, before nav, before `<main>` — rendered in root layout
- Non-dismissable: No close button, no toggle, no collapse
- Required copy (verbatim minimum): "This is an experimental astrological model. Earthquakes cannot be reliably predicted. This tool is for research purposes only and should not be used for safety decisions."
- Styling: `bg-amber-50 border-b border-amber-200 px-4 py-3 text-sm text-amber-900`
- Bold lead-in: **Disclaimer:** prefix in `font-semibold`

## Layout Contract

### Main Calendar Page
```
┌─────────────────────────────────────────────────────────┐
│  DISCLAIMER BANNER (amber, full width, non-dismissable)  │
├─────────────────────────────────────────────────────────┤
│  NAV: "Earthquake Astrology 2026"      [Methodology]     │
├─────────────────────────────────────────────────────────┤
│  PAGE CONTENT (main)                                      │
│  ┌──────────────────────────────────────────────┐  ┌───┐ │
│  │  10 MonthGrids (scrollable, single column    │  │ D │ │
│  │  on mobile; 2-col on md+)                    │  │ e │ │
│  │                                              │  │ t │ │
│  │  Each MonthGrid:                             │  │ a │ │
│  │    Month label (h2)                          │  │ i │ │
│  │    7-col grid: Su Mo Tu We Th Fr Sa          │  │ l │ │
│  │    Date cells (aspect-square, rounded)       │  │   │ │
│  │      white = normal                          │  │ P │ │
│  │      orange = high-risk                      │  │ a │ │
│  └──────────────────────────────────────────────┘  │ n │ │
│                                                     │ e │ │
│                                                     │ l │ │
│                                                     └───┘ │
└─────────────────────────────────────────────────────────┘
```

- Mobile (<768px): Single column, 10 months stacked; detail panel overlays full screen at `w-full`
- Desktop (≥768px): Main calendar area has `pr-0` when panel is closed; adds `lg:pr-96` when panel is open to prevent content shift. Panel is `fixed right-0 w-96`.
- Calendar grid gap: `gap-1` between cells
- Month section bottom margin: `mb-8` (32px)
- Page padding: `px-4 py-8` on mobile, `px-8 py-8` on desktop

### Methodology Page
- Single column, max-width `max-w-2xl mx-auto`, `px-4 py-8`
- Metrics table: full-width, alternating row background (`even:bg-gray-50`)
- Confusion matrix: 2×2 grid or 4 labeled cells (TP/FP/FN/TN)
